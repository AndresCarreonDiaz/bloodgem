# Research: elevenlabs-audio

## Summary

ElevenLabs offers two separate generation APIs relevant to Bloodgem, both authenticated with a single `xi-api-key` header and both returning raw binary audio (default MP3) rather than JSON-wrapped payloads. Sound effects are generated via `POST https://api.elevenlabs.io/v1/sound-generation` with a JSON body of `text` (required), `duration_seconds` (0.5-30s, null = model auto-picks), `prompt_influence` (0-1, default 0.3), `loop` (boolean, default false), and `model_id` (default `eleven_text_to_sound_v2`); the output format is a query parameter (e.g. `?output_format=mp3_44100_128`). Seamless looping IS supported natively: setting `loop: true` on the v2 model produces a sound whose ending blends into its beginning with no gap — the documented pattern for game ambience is "generate 30 seconds of soft rain, then loop it endlessly."

Music is generated via `POST https://api.elevenlabs.io/v1/music` (plus `/v1/music/stream` for chunked streaming, `/v1/music/detailed` for audio+metadata, and a free `POST /v1/music/plan` endpoint that drafts a composition plan without costing credits). You supply EITHER a `prompt` (with optional `music_length_ms`, 3,000-600,000 ms, and `force_instrumental: true`) OR a `composition_plan` — a JSON object of up to 30 `chunks`, each with `text` (section label/lyrics/inline cues like `{guitar solo}`), `duration_ms` (3,000-120,000), `positive_styles`/`negative_styles` (up to 50 each, English only), and `context_adherence` (low/medium/high). Instrumental-only is explicitly supported two ways: the `force_instrumental` boolean (prompt mode) or writing "instrumental only" / "completely instrumental" into the prompt. Use `model_id: "music_v2"` — `music_v1` is the legacy default and v2 has better prompt adherence and section-by-section composition.

Pricing: the API pricing page lists Eleven Music at $0.15/minute and Sound Effects at $0.12/minute; in credit terms the docs state SFX cost 40 credits per second when you specify `duration_seconds` (auto-duration is billed as a flat per-generation fee, reported at ~200 credits; older/plan-dependent figures of 100 credits/gen and 20 credits/sec also circulate — the help-center article is behind a 403 so I could not confirm the current flat rate first-hand). Plan inclusions: Creator ($22/mo) covers roughly 40 minutes of music and ~150 SFX generations — comfortably a full asset pass for a game like Bloodgem. The composition-plan endpoint costs zero credits, so you can iterate on boss-theme structure for free and only pay when you render audio. One caveat for a Bloodborne-inspired project: the music API actively rejects copyrighted artist/soundtrack references with a `bad_prompt` / `bad_composition_plan` error (it returns a `prompt_suggestion` field), so describe the gothic sound ("dark gothic orchestral, Latin choir, dissonant strings, tolling bells") instead of naming Bloodborne or its composers.

## Key findings

- SFX endpoint: POST https://api.elevenlabs.io/v1/sound-generation with header `xi-api-key: <key>` and Content-Type: application/json. Body fields per the API reference: `text` (string, required — "The text that will get converted into a sound effect"), `model_id` (string, default `eleven_text_to_sound_v2`), `duration_seconds` (number|null, range 0.5-30, null = auto-determined), `prompt_influence` (number|null, 0-1, default 0.3 — "Controls adherence to prompt"), `loop` (boolean, default false — "Whether to create a sound effect that loops smoothly"). `output_format` is a QUERY parameter, not a body field (e.g. mp3_44100_128 default, mp3_22050_32, pcm_16000, opus_48000_64; 24+ codec_samplerate_bitrate variants including mp3, pcm, opus, u-law). Response 200 is raw binary audio (application/octet-stream); 422 = validation error. (elevenlabs.io/docs/api-reference/text-to-sound-effects/convert)
- Seamless looping: YES for sound effects — `loop: true` (v2 model only) creates "a seamless loop, with the ending blending into the beginning without a noticeable gap"; the docs' canonical use case is generating 30s of ambience (e.g. 'soft rain') and looping it endlessly for games/film. Note the docs say WAV 48kHz output is available for NON-looping effects; looped effects come as MP3. (docs sound-effects capability page)
- Working SFX curl (from docs): curl -X POST https://api.elevenlabs.io/v1/sound-generation -H "xi-api-key: $ELEVENLABS_API_KEY" -H "Content-Type: application/json" -d '{"text": "Spacious braam suitable for high-impact movie trailer moments"}' — add e.g. '{"text":"...","duration_seconds":2,"prompt_influence":0.8,"loop":true}' and ?output_format=mp3_44100_128 --output sfx.mp3 for full control.
- SFX credit cost: docs state "40 credits per second when duration is specified" (so a 1s sword impact = 40 credits, a 30s ambience loop = 1,200 credits); auto-duration is a flat per-generation fee reported at ~200 credits (older/plan-dependent sources say 100 credits/gen and 20 credits/sec). API dollar rate: $0.12/minute (elevenlabs.io/pricing/api). Help-center pricing article returns HTTP 403 so the flat rate could not be verified first-hand.
- Music endpoints: POST https://api.elevenlabs.io/v1/music (compose, returns binary audio), POST /v1/music/stream (same params, chunked audio, paid plans only), POST /v1/music/detailed (returns composition plan + metadata + audio + optional song_id), POST /v1/music/plan (a.k.a. composition-plan — generates a plan JSON from a prompt, "does not cost any credits but is subject to rate limiting"), plus enterprise-only /v1/music/upload and /v1/music/video-to-music. Auth: same xi-api-key header. Music API "requires paid account access."
- Music compose body fields (API reference): `prompt` (string|null) XOR `composition_plan` (object|null) — cannot combine; `music_length_ms` (integer, 3,000-600,000 ms, only valid with prompt; model picks length if omitted); `model_id` (default `music_v1`, options music_v1|music_v2 — use music_v2); `seed` (integer, not usable with prompt); `force_instrumental` (boolean, default false, prompt mode only); `respect_sections_durations` (boolean, default true, plan mode only, ignored by music_v2); `store_for_inpainting` (boolean, default false); `sign_with_c2pa` (boolean, default false, MP3 only). Query param `output_format` default "auto": mp3_44100_128 (v1 default), mp3_48000_192 (v2 default), mp3_48000_240, mp3_48000_320, pcm_44100, opus_48000_96/128.
- Instrumental-only music: YES, two mechanisms — `force_instrumental: true` in the request body (prompt mode), or phrases like "instrumental only" / "completely instrumental" in the prompt text; the best-practices doc also documents timing control like "instrumental only after 1:45" and "lyrics begin at 15 seconds".
- Composition plan structure (music_v2): {"chunks": [{"text": "[Section label]\nlyrics + inline cues in braces like {guitar solo}", "duration_ms": 3000-120000, "positive_styles": [up to 50, English only], "negative_styles": [up to 50], "context_adherence": "low|medium|high" (default high)}]} — max 30 chunks, total duration 3 s to 10 min; first chunk's styles establish overall tone/genre. By default the model strictly follows duration_ms values. (docs composition-plans guide + github.com/elevenlabs/skills music api_reference.md)
- Length limits discrepancy worth knowing: the API reference allows music_length_ms up to 600,000 ms (10 min) and composition plans state "total 3 seconds to 10 minutes", while the older capability overview page says "maximum duration of 5 minutes" — the 10-minute figure matches the current API schema.
- Music cost: $0.15/minute at API dollar rates (elevenlabs.io/pricing/api), plus $1.50 per finetune. Plan inclusions from the same page: Starter $6/mo = 3 min music + ~8 SFX generations; Creator $22/mo = 40 min music + ~150 SFX; Pro $99/mo = 147 min + ~605 SFX. Third-party sources put the credit equivalent around 900 credits per music minute (help-center article 403'd, unverified).
- Copyright guardrails: music API returns `bad_prompt` (with a `prompt_suggestion` field) when the prompt references copyrighted artists/bands/lyrics, and `bad_composition_plan` (with `composition_plan_suggestion`) for copyrighted styles in plans — so 'sounds like the Bloodborne OST' will be rejected. Common errors: 401 invalid key, 422 invalid params, 429 rate limit. (github.com/elevenlabs/skills music api_reference.md)
- SFX prompting guide (official docs): the model understands audio-production terminology — documented terms include "impact", "whoosh", "ambience", "one-shot", "loop", "stem", "glitch", "drone", and "braam: big, brassy cinematic hit that signals epic or dramatic moments". Example prompts from docs/skills repo: "Cinematic Braam, Horror" (the official quickstart example), "Glass shattering on concrete", "Footsteps on gravel, then a metallic door opens" (sequenced multi-event SFX), "90s hip-hop drum loop, 90 BPM", "8-bit retro jump sound". Guidance: high prompt_influence for explicit briefs, low for atmospheric variation.
- Music prompting guide (official best-practices doc): abstract mood words ('eerie', 'foreboding') work as well as technical language; the model "accurately follows BPM and often captures the intended musical key" — include markers like "130 BPM" and "in A minor"; use "solo" before an instrument for isolation; prompt length does not correlate with quality (third-party fal.ai guide: 20-60 words optimal). fal.ai's dark-trailer example prompt: "Extremely dark, tense and powerful, cinematic sound design, electronic hybrid, trailer music, evil, braam, braam horns, impacts, boom, rising tension, completely instrumental".
- Official SDKs: `@elevenlabs/elevenlabs-js` (npm) — `await elevenlabs.textToSoundEffects.convert({ text: "Cinematic Braam, Horror" })` and `await elevenlabs.music.compose({ prompt, musicLengthMs: 10000, modelId: "music_v2" })` (note camelCase in the JS SDK vs snake_case in raw HTTP). Python: `elevenlabs` package with `client.music.compose(...)` and `client.text_to_sound_effects.convert(...)`.
- Working music curl (from elevenlabs/skills repo): curl -X POST "https://api.elevenlabs.io/v1/music" -H "xi-api-key: $ELEVENLABS_API_KEY" -H "Content-Type: application/json" -d '{"prompt": "An upbeat electronic track with synth leads", "music_length_ms": 30000, "model_id": "music_v2"}' --output output.mp3 — add "force_instrumental": true for score-only tracks.
- Licensing: "Eleven Music is cleared for nearly all commercial uses, from film and television to podcasts and social media videos, and from advertisements to gaming" — gaming is explicitly named; exact rights vary by subscription tier per their music terms page.

## Actionable for Bloodgem

- Build a one-time TypeScript asset-generation script (scripts/generate-audio.ts using @elevenlabs/elevenlabs-js) that batch-generates and caches all SFX/music into assets/audio/ — never call the API at game runtime; commit the MP3s and treat ElevenLabs as a build-time tool.
- For combat SFX, always pass explicit duration_seconds to control cost and timing: 0.5-1.0s for hits/parries (40 credits each at 1s), 1.5s for dodge whooshes, 2-3s for monster roars. Use prompt_influence 0.7-0.9 for precise one-shots (UI clicks, weapon impacts) and the 0.3 default for ambience so retries give useful variation.
- Use loop: true + duration_seconds: 20-30 for ambient beds (e.g. "Low howling wind through gothic stone streets, distant tolling church bell, creaking wood, horror ambience, loop") and play them via Web Audio AudioBufferSourceNode with .loop = true — the API guarantees gapless loop points so you avoid manual crossfade code in Canvas/Web Audio.
- Concrete SFX prompt patterns for Bloodgem, using the terminology the model is trained on: impacts — "Heavy wet flesh impact with bone crunch, visceral, one-shot"; whooshes — "Fast metallic whoosh of a serrated blade slicing air, sharp transient"; monster roars — "Guttural inhuman beast roar, layered snarl, cathedral reverb tail, horror"; parry — "Sharp metallic clang with ringing tail, gunshot crack"; use "braam" for boss-intro stingers.
- For music, always send model_id: "music_v2" and force_instrumental: true. A tuned prompt for the game's exploration theme: "Dark gothic orchestral, slow dissonant strings, deep male choir chanting in Latin, church organ, tolling bells, sparse and mournful, 70 BPM in D minor, completely instrumental" (20-60 words, BPM + key included per best practices).
- Structure boss themes with composition plans (free to draft via POST /v1/music/plan, zero credits): chunk 1 = tense intro (duration_ms 15000, positive_styles ["low string ostinato","distant choir"]), chunk 2 = phase-1 combat (45000, ["full orchestra","aggressive percussion","frantic violins"]), chunk 3 = phase-2 frenzy (45000, ["pounding taiko","screaming brass","dissonant choir stabs"]) — then trigger phase transitions in-game by seeking/crossfading between the known chunk offsets, since you control exact duration_ms per section.
- Never mention Bloodborne, FromSoftware, or composer names in music prompts — the API returns bad_prompt/bad_composition_plan for copyrighted references; describe the aesthetic instead (gothic, Victorian horror, tragic choir, hunter's dirge). If rejected, read the returned prompt_suggestion field and retry with it.
- Request output_format=mp3_44100_128 for SFX (small files, fine for game one-shots) and mp3_48000_192 for music; for web delivery weight, consider opus_48000_96 for ambience loops since HTMLAudioElement/Web Audio decode Opus in all modern browsers Bloodgem targets.
- Budget on the Creator plan ($22/mo): ~40 min of music (about 20-25 boss/area themes at 90-120s each) plus ~150 SFX generations — one month covers a complete audio pass; expect roughly 3 retries per sound, so script your prompts and review in batches rather than generating interactively.
- Generate music at 60-120s and loop at the track level in-game (music_length_ms: 90000 typical); the music API has no loop parameter (only SFX does), so either author intro+loopable-body via a composition plan whose last chunk resolves back to the first chunk's texture, or add a short crossfade (~500 ms) in your Web Audio music player.
